# Bloom by BotaniK — Master Reference

> Document de référence centrale. Toujours lire en premier.

## Identité du projet

- **Nom** : Bloom by BotaniK
- **Mission** : Reconstruire un savoir utile, rigoureux et accessible sur les plantes
- **Pôle** : Science botanique + IA agentique + Production de contenu
- **Langue** : Français (contenu) / Anglais (structure technique)
- **Version** : 1.0.0
- **Date de création** : 2026-05-29

## Doctrine fondamentale

### 1. La plante est l'objet central
Chaque information, chaque contenu, chaque décision tourne autour d'une fiche plante validée.

### 2. Preuve avant claim
Aucun claim santé sans référence scientifique validée. Voir `05_politique_preuves_et_claims.md`.

### 3. Le savoir est vivant
Chaque fiche est mise à jour régulièrement par l'Agent Veilleur Botanique.

### 4. L'IA est un amplificateur, pas un auteur
Les agents IA structurent, vérifient et produisent. L'humain valide et décide.

## Bibliothèque cible

- **148 plantes** identifiées (voir `04_bibliotheque_148_plantes.md`)
- Catégories : adaptogènes, immunomodulateurs, anti-inflammatoires, nootropiques, digestives

## Agents et responsabilités

| Agent | Responsabilité |
|-------|----------------|
| Agent Maître | Orchestration, validation finale, gestion des conflits |
| Agent Veilleur Botanique | Collecte, mise à jour, veille PubMed/Cochrane |
| Agent Validateur Scientifique | Validation des preuves, niveau d'évidence, claims |
| Agent Archiviste Éditorial | Production contenu, fiches plantes, articles |

## Standards de qualité

- Niveau d'évidence minimum : étude clinique humaine ou méta-analyse
- Format de citation : APA 7e édition
- Mise à jour des fiches : tous les 6 mois minimum
- Relecture humaine : obligatoire avant publication
